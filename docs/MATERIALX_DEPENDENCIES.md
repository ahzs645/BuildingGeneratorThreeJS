# MaterialX dependency and license notes

## Shipped runtime

- Three.js 0.185.1 is installed from npm and licensed MIT. Its `MaterialXLoader`, `WebGPURenderer`, and TSL modules are imported from the dependency; none are copied into this repository.
- Three's MaterialX node implementation carries an ASWF MaterialX notice and Apache-2.0 terms in `node_modules/three/src/nodes/materialx/DISCLAIMER.md`. This prototype imports that implementation and does not copy or modify its source.
- Blender 5.1.2 is invoked as an external extraction/rendering tool. Blender code is not linked, copied, or distributed here. The `.mtlx`, JSON, `scene-contract.json`, and Blender reference PNGs are generated from the supplied project material.
- The extraction script uses Python's standard library plus the `bpy`, `pxr`, and `MaterialX` modules bundled with Blender.
- `studio-environment.exr` is a repository-generated linear test fixture. It contains no photographed or third-party environment imagery and is regenerated by the Blender reference script.
- The three web direct-light PNGs and coordinate-cardinal PNG are renderer outputs of repository-owned fixtures. They contain no third-party imagery. The corresponding Blender PNGs and scene contract are external-tool outputs, not copied Blender source.
- The baked normal/roughness maps are outputs derived from the supplied `.blend`. Blender/Cycles is invoked externally; no Blender source or add-on code is copied into the repository.

## Exporter evaluation

Blender 5.1 native USD export was selected because it is installed, reproducible, exports a MaterialX-flavored USDShade network, handles texture copying, and has no additional code-distribution requirement. It does not emit standalone `.mtlx`, so the repository-owned script performs a small, node-type-based USDShade-to-MaterialX XML conversion and validates the result.

`bhouston/blender_materialx_addon` was evaluated at commit `bbb33506e15a2a5dede6755a015a98b6d3fb3703`. Its README claims MIT and advertises direct `.mtlx`, texture export, Noise, Wave, and Bump mappings. At that commit the repository root has no `LICENSE` file even though the README links to one; the bundled `materialx_addon/mtlxutils/LICENSE` covers that subcomponent under Apache-2.0. Because the top-level licensing grant is incomplete and Three still lacks the add-on's `wave`/`bump` categories, no add-on code was copied, installed, or vendored.

No GPL exporter code is vendored. If a future team chooses an external GPL Blender add-on, keep it as a separately installed Blender tool, record its version, and review distribution obligations before placing any of its code in this repository.

## Upstream Three.js parity evaluation

The optional `npm run materialx:test:upstream` experiment evaluates open Three.js pull request [#33485](https://github.com/mrdoob/three.js/pull/33485) at pinned commit `bce55b294825d273eae3e178aab3191f719594e6`. The script downloads the MIT-licensed fork through `npm pack` into the ignored `.cache/materialx/three-pr33485` directory. The downloaded package retains Three's `LICENSE`; none of its code or build products are committed or distributed by this repository.

The PR's checked-in `build/three.tsl.js` is stale relative to its MaterialX loader at the pinned commit and lacks `mx_cell_noise_vec3`. The isolated Vite configuration therefore resolves the package's `src/Three.WebGPU.js`, `src/Three.TSL.js`, and `src/Three.js` source entrypoints. This mirrors the Material Fidelity Suite's build-from-source evaluation and is enabled only when `MATERIALX_THREE_ROOT` is set. Normal builds continue to resolve npm Three 0.185.1.

## Official MaterialX shader-generator evaluation

The official MaterialX 1.39.4 JavaScript release was evaluated at commit `270b5cf2ae2be24a3b6ef4b0569f1c93038dda1d` under Apache-2.0. Its GenShader JavaScript/WASM/data payload was used only in `/tmp`; no runtime is vendored.

Blender 5.1.2 also bundles the official MaterialX 1.39.4 Python GenShader/GenGlsl modules. The repository generates ESSL offline and `npm run materialx:smoke:essl` compiles and links every committed program in headless WebGL2. This removes the approximately 4.1 MB browser generator payload from the proposed architecture.

The upstream MaterialX 1.39.4 Apache license and third-party notices are committed at `public/materialx/licenses/LICENSE` and `public/materialx/licenses/THIRD-PARTY.md`. Generated shaders retain embedded OSL notices. No MaterialX generator runtime or Blender GPL source is distributed.

MaterialX's generated Noise implementation emits the applicable Sony Pictures Imageworks/Open Shading Language BSD-style notice inside the fragment shader. That notice must not be stripped during minification or packaging.

The generated shaders are not automatically compatible with Three's lights or PMREM uniforms. They require a project-owned adapter for MaterialX light data and radiance/irradiance environment resources, kept inside the isolated lab boundary.

The committed research screenshot is rendered output, not copied source. The evaluated release, instructions, and notices are linked below.

## Adapter and feasibility source audit

| Source | Observed license | Use in this project |
| --- | --- | --- |
| ASWF MaterialX 1.39.4, including Web Viewer and prefilter generator | Apache-2.0 plus recorded third-party notices | Safe foundation; retain notices |
| Three.js and PR #33485 | MIT | Safe to import from the installed dependency or evaluate at a pinned commit |
| Material Fidelity | MIT | Safe methodology and fixtures subject to its license |
| MaterialXNodeDocs, commit `8c55f105dc6c60a12880afbf284f3a29a30ba14f` | Apache-2.0 | Safe reference implementation; reimplement the minimal Three 0.185.1 adapter rather than vendor its r128/WASM application |
| `@needle-tools/materialx` 1.7.4 | PolyForm Noncommercial 1.0.0 | Inspect and test for feasibility only; do not ship or copy wrapper source without separate permission |
| Blender Eevee 5.1 shader sources | GPL-2.0-or-later | Inspect behavior and compare results only; do not copy shader source into the web runtime |
| Langenium | MIT repository, but mixed/unclear provenance in relevant shader fragments | Architecture and visual evidence only; do not copy shader chunks |
| `bhouston/blender_materialx_addon`, commit `bbb33506e15a2a5dede6755a015a98b6d3fb3703` | README says MIT; repository still has no root `LICENSE` | Do not vendor; native Blender extraction remains selected |
| `Bentschi/Blender-GLSL-translator` | No license | Inspect feasibility only; no copying |
| `netherby/blender-material-to-glsl` | GPL-3.0 | No copying into this project; any use would require a separate compliance decision |

An absent or restrictive license does not prevent observing public behavior or independently implementing documented interfaces. It does prevent treating that source as reusable implementation material. The feasibility conclusion does not depend on those sources: the official Apache MaterialX viewer, Apache MaterialXNodeDocs adapter, MIT Three changes, and the already successful offline compile provide a complete permissively licensed basis.
